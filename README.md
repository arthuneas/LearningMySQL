##  Esquema do Repositório 

Este projeto não é apenas uma coleção de scripts; é um laboratório de persistência. Aqui, exploramos como transformar caos em dados estruturados através da álgebra relacional.

### Estrutura de Domínio (Learning Modules)

| Camada | Função Principal | Comandos Chave |
| :--- | :--- | :--- |
| **DQL** | Extração de Inteligência | `SELECT`, `JOIN`, `GROUP BY` |
| **DML** | Manipulação de Registros | `INSERT`, `UPDATE`, `DELETE` |
| **DDL** | Definição de Estrutura | `CREATE`, `ALTER`, `DROP` |

---

##  Relacionamentos e Integridade

O foco principal do estudo recai sobre a **Integridade Referencial**. O objetivo é dominar a arte de conectar entidades sem gerar redundâncias desnecessárias:

> **Normalização:** Aplicação rigorosa da 1ª, 2ª e 3ª Formas Normais para garantir bancos de dados performáticos e escaláveis.
>
> **Cardinalidade:** Mapeamento preciso de relações `1:1`, `1:N` e `N:N` (Tabelas Intermediárias).



## Stack de Gerenciamento

* **MySQL Workbench:** Engenharia reversa e diagramação.
* **Docker:** Orquestração de instâncias de banco de dados para ambientes de teste.
